# amo

### 1. Создание данных: data_creation.py
  - Генерирует данные о температуре. Создаёт 5 наборов данных (datasets_count=5), 3 из которых сохраняются в папку train, а остальные 2 в test.
  - Температура меняется циклично (использована функция sin). Колебание от +7 до -7 вокруг средней температуры 15.
  - np.random.normal добавляет реалистичные небольшие колебания погоды (шумы).
  - С помощью вероятности (ANOMALY_CHANCE) случайно выбираются дни, когда происходит резкое изменение погоды (анамалии).

### 2. Предобработка данных: model_preprocessing.py
  - Извлекает признаки: преобразует столбец date в числовые признаки: day_of_year (номер дня в году), month и day_of_month.
  - Нормализует столбец temperature с помощью StandardScaler: приводит среднее значение к 0, а стандартное отклонение к 1.
  - Сохраняет обработанные данные обратно в CSV-файлы, перезаписывая старые.
  - Применяется и к тренировочным, и к тестовым данным

### 3. Подготовка и обучение модели: model_preparation.py
  - Объединяет все файлы из папки train в один большой массив данных
  - Создаёт последовательности для обучения: использует данные за 30 дней (SEQ_LENGTH), чтобы предсказать температуру на 31-й
  - Для прогнозирования временных рядов используется рекуррентная нейронная сеть (RNN), а конкретно — LSTM
  - Архитектура: Один слой LSTM с 50 нейронами считывает последовательность, а слой Dense(1) выдает одно число — прогноз температуры на следующий день
  - Валидация: Часть тренировочных данных (test_size=0.2) отделяется для проверки качества обучения после каждой эпохи (чтобы увидеть, не переобучается ли модель)
  - Обучает модель с помощью оптимизатора adam и функции потерь mse
  - Сохраняет итоговую обученную модель в файл trained_model.h5

### 4. Тестирование модели: model_testing.py
  - Загружает сохранённую модель из файла trained_model.h5
  - Для каждого файла в папке test формирует последовательности данных так же, как при обучении
  - Прогнозирует температуру и вычисляет метрику MSE (среднеквадратичная ошибка) для каждого файла
  - Выводит среднюю ошибку MSE по всем тестовым наборам данных (Низкое значение MSE говорит о хорошем качестве модели)

### 5. Bash-скрипт: pipeline.sh
  - последовательно выполняет все шаги
    - генерирует данные (`python data_creation.py`)
    - делает их предобработку (`python model_preprocessing.py`)
    - обучает модель (`python model_preparation.py`)
    - тестирует её (`python model_testing.py`)

### Как использовать:
  - Откройте терминал.
  - Перейдите в директорию с файлами проекта (команда cd путь).
  - Сделайте скрипт исполняемым: chmod +x pipeline.sh.
  - Запустите весь проект одной командой: ./pipeline.sh.
  - Вы увидите вывод в консоль, который подтверждает прохождение каждого шага. Готово!
